# at2someone

> 灵感来源于某服务器五周目（2021）时的一个（我很喜欢但是忘了他叫啥名字的）Sponge 插件。
>
> 因为太过于思念，所以写了一个类似功能的 Bukkit 插件。权当 Java 练手 欢迎大佬批评。

一个可以在 bukkit 服务器内 @人的插件。在聊天栏中 `@ + 用户名` 或直接 `输入用户名` 即可在公屏中提及某人。且附带勿扰模式和显示昵称等原先没有的功能。

PS:当isprefix关闭的时候 玩家可以在句内提及人 如 `你好qixuanjun233`

## 兼容性

1. Paper / Purpur 服务端 (必须支持 Paper API)
2. 游戏版本 ≥ 1.19.4

**注意**：为了获得完美的 `@` 补全体验，本插件已升级架构，需要 Minecraft 1.19.4 或更高版本的服务端支持。旧版本 (1.18) 用户建议升级服务端或使用本插件的旧版本。

经测试插件可运行于 Paper 1.20.4+、Leaves 1.20.6 。目前仅支持中文(有计划制作多语言版本)。

## 权限

| 权限         | 默认是否可用          | 解释                                |
| ------------ | --------------------- | ----------------------------------- |
| `at.mention` | `玩家:是`<br/>`OP:是` | 玩家**必须拥有此权限才可提及他人**  |
| `at.dnd`     | `玩家:是`<br/>`OP:是` | 玩家开关勿扰模式的权限              |
| `at.atall`   | `玩家:否`<br/>`OP:是` | 玩家**必须拥有此权限才可@全体成员** |
| `at.admin`   | `玩家:否`<br/>`OP:是` | 插件管理权限（**危险**）            |

###### Tips:您可通过禁用某玩家或Group的`at.mention`权限实现 启用/禁用 此玩家/群组@人的权限，`at.atall`同理。

## 指令

![](https://s41.ax1x.com/2025/12/20/pZ3PwfP.png)

| 指令                | 权限                | 用途                                  |
| ------------------- | ------------------- | ------------------------------------- |
| /at                 | `N/A`               | 可用指令+插件状态查询                 |
| /at dnd on/off      | `at.dnd (默认就有)` | 勿扰模式(可见下方解释)                |
| /at isprefix on/off | `at.admin`          | 是否需要输入@才能提及人(可见下方解释) |
| /at atall on/off    | `at.admin`          | 是否允许@全体成员(可见下方解释)       |
| /at enable          | `at.admin`          | 使能插件功能                          |
| /at disable         | `at.admin`          | 禁用插件功能                          |
| /at reload          | `at.admin`          | 重载插件配置文件                      |

## 功能

#### 勿扰模式

![](https://s41.ax1x.com/2025/12/20/pZ3P6mQ.png)

当玩家输入 `/at dnd on` 时，玩家将**不会收到当其他玩家提及时所发出的提示音与标题**，仅会在聊天栏中高亮用户名。

此模式可通过 `/at dnd off` 进行关闭

#### isPrefix

![](https://s41.ax1x.com/2025/12/20/pZ3Pr6S.png)

当 OP 输入 `/at isprefix on` 时，玩家必须输入@+用户名才可提及某人。

当 OP 输入 `/at isprefix off` 时，玩家仅需输入用户名即可提及某人(如图左下角,默认为此)。

## at全体成员

![](https://s41.ax1x.com/2025/12/20/pZ3PsOg.png)

当 OP 输入 `/at atall on` 时，拥有`at.atall`权限的玩家可@全体成员。

当 OP 输入 `/at atall off` 时，玩家不可以@全体成员。
